TCP/IP - Передача пакетів в мережі здійснюється відповідно до правил - стандартами, протоколами. 
До них відносяться протоколи TCP / IP і UDP (TCP - transmission control protocol, IP - Internet protocol, UDP - user datagram protocol). 
У мережах, заснованих на використанні набору протоколів TCP / IP, інформація передається пакетами зі стандартизованої структурою. 
Ці пакети називаються IP-дейтаграммами (IP Datagram)

HTTP - протокол перенесення гіпертексту чи протокол передачі гіпер-текстових документів (англ. HyperText Transfer Protocol (HTTP) — це основний протокол, 
що дозволяє передавати документи гіпермедіа в мережі, в основному між переглядачем та сервером, щоб люди могли їх читати. 
Поточна версія специфікації HTTP називається HTTP/2

DNS (англ. Domain Name System - система доменних імен) - комп'ютерна розподілена система для отримання інформації про домени. 
Найчастіше використовується для отримання IP-адреси по імені хоста (комп'ютера або пристрою), 
отримання інформації про маршрутизацію пошти та про обслуговуючі вузли для протоколів у домену.
DNS використовується в першу чергу для перетворення символьних імен в IP-адреси, або навпаки.

Hypertext Markup Language (HTML) (укр. Мова розмітки гіпертекстових документів) — стандартна мова розмітки для створення веб-сторінок і веб-додатків.
HTML-документ складається з елементів: теги (початковий та кінцевий), атрибути та зміст (контент).
Тег — це назва елементу, записана у кутових дужках (< >). Атрибути задають технічну інформацію про елемент. 
Зміст елементу — це вся необхідна текстова та графічна інформація документу, яка буде відтворюватися браузером на екрані
